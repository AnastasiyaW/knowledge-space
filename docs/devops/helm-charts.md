# Helm Charts

Kubernetes package manager: templating, versioning, and managing complex deployments.

## Key Facts

- Helm = package manager for Kubernetes; a **chart** = package of K8s resource templates
- **Release** = installed instance of a chart; same chart can be installed multiple times with different names
- `values.yaml` = default configuration; override with `-f custom-values.yaml` or `--set key=value`
- Chart structure: `Chart.yaml` (metadata), `values.yaml` (defaults), `templates/` (K8s manifests with Go templating)
- Helm uses Go template syntax: `{{ .Values.replicas }}`, `{{ .Release.Name }}`, `{{ include "chart.name" . }}`
- **Helm repositories** host chart packages; `helm repo add` to register, `helm repo update` to refresh
- Helm 3 removed Tiller (server-side component); stores releases as Secrets in the namespace
- **Chart dependencies** defined in `Chart.yaml` under `dependencies:`; fetched with `helm dependency update`
- **Hooks** = resources that run at specific lifecycle points (pre-install, post-upgrade, pre-delete)
- [[kubernetes-pods]] are the ultimate output of chart templates
- [[kubernetes-services]] and Ingress commonly templated in charts
- Helm supports **rollback** to any previous revision: `helm rollback release-name revision`

## Patterns

```
# Chart directory structure
mychart/
  Chart.yaml          # name, version, appVersion, dependencies
  values.yaml         # default values
  charts/             # dependency charts
  templates/
    deployment.yaml
    service.yaml
    ingress.yaml
    configmap.yaml
    _helpers.tpl       # named template definitions
    NOTES.txt          # post-install instructions
```

```yaml
# Chart.yaml
apiVersion: v2
name: myapp
version: 0.1.0          # chart version
appVersion: "1.2.0"     # app version
dependencies:
- name: postgresql
  version: "12.x.x"
  repository: "https://charts.bitnami.com/bitnami"
  condition: postgresql.enabled

# values.yaml
replicaCount: 3
image:
  repository: myapp
  tag: "1.2.0"
  pullPolicy: IfNotPresent
service:
  type: ClusterIP
  port: 80
ingress:
  enabled: true
  host: app.example.com
resources:
  requests:
    cpu: 100m
    memory: 128Mi
  limits:
    cpu: 500m
    memory: 512Mi

# templates/deployment.yaml (excerpt)
# spec:
#   replicas: {{ .Values.replicaCount }}
#   template:
#     spec:
#       containers:
#       - name: {{ .Chart.Name }}
#         image: "{{ .Values.image.repository }}:{{ .Values.image.tag }}"
#         resources: {{ toYaml .Values.resources | nindent 12 }}
```

```bash
# Repository management
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo update
helm search repo nginx

# Install / Upgrade
helm install my-release ./mychart
helm install my-release bitnami/nginx -f prod-values.yaml
helm upgrade my-release ./mychart --set replicaCount=5
helm upgrade --install my-release ./mychart   # install if not exists

# Inspect before applying
helm template my-release ./mychart            # render templates locally
helm install my-release ./mychart --dry-run   # server-side dry run
helm diff upgrade my-release ./mychart        # requires helm-diff plugin

# Release management
helm list                              # list releases
helm status my-release                 # release info
helm history my-release                # revision history
helm rollback my-release 2             # rollback to revision 2
helm uninstall my-release              # remove release

# Dependency management
helm dependency update ./mychart
helm dependency build ./mychart
```

## Gotchas

- `helm upgrade` without `--install` fails if release doesn't exist; use `helm upgrade --install` for idempotency
- Hook resources (Jobs) are NOT garbage collected by default; add `helm.sh/hook-delete-policy: hook-succeeded`
- Values precedence: `--set` > `-f custom.yaml` > `values.yaml`; `--set` values are strings - use `--set-json` for complex types
- Template whitespace issues: use `{{-` and `-}}` to trim whitespace; `nindent` for proper YAML indentation
- Chart version (`version:`) must follow SemVer and be bumped for each change; `appVersion` is informational
- `helm template` doesn't validate against cluster; some resources may template fine but fail on `kubectl apply`
- Secrets in values.yaml are stored as base64 in K8s Secrets; consider helm-secrets or external-secrets-operator
